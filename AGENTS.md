# Interaction

- Confirm if my requirements are clear.
- If something is unclear, stop, name it, and ask.
- If multiple interpretations exist, present them. Do not pick silently.
- State information directly.
- Skip rhetorical framing.
- If a simpler approach exists, say so.

# Coding

- Never expose PII or confidential information in tracked files, examples,
  docs, config, logs, generated output, or git history.
- Treat internal endpoints, email addresses, usernames, local machine paths,
  private repo names, and account-specific identifiers as sensitive by default.
- Use environment variables or local-only ignored files for sensitive values.
